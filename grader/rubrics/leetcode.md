# LeetCode Grading Rubric

Grade the solution on these 5 dimensions (1–5 each):

## 1. Correctness (1–5)
- 5: Handles all cases including edge cases
- 3: Correct for main cases, misses some edge cases
- 1: Wrong answer or major logic error

## 2. Time Complexity (1–5)
- 5: Optimal complexity with correct justification
- 3: Correct but not optimal
- 1: Wrong complexity analysis or brute force when better exists

## 3. Space Complexity (1–5)
- 5: Optimal space with correct justification
- 3: Acceptable but improvable
- 1: Unnecessary memory usage

## 4. Code Quality (1–5)
- 5: Clean, readable, good names, no redundancy
- 3: Works but messy or unclear
- 1: Hard to read, poor names, unnecessary complexity

## 5. Edge Cases (1–5)
- 5: Explicitly handles empty input, single element, negatives, duplicates
- 3: Some edge cases considered
- 1: No edge case handling

## Output format
Return a markdown report with:
- `## Grading Report` as the title
- `Grade: X/10` immediately under the title
- `Take-home: <one sentence>` immediately under the grade
- Score per dimension
- Total score /25
- Specific improvement suggestions
- Optimal solution if the submitted one is suboptimal
